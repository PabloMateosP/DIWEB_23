Principios Pautas Criterios de Conformidad A Técnicas Suficientes
Perceptible
1.1 - Proporcionar alternativas textuales
1.2 - Medios tempodependientes
1.3 - Crear contenido que pueda presentarse de diferentes formas
1.4 - Facilitar a los usuarios ver y oír el contenido
1.1.1 Contenido no textual A -> Todo contenido no textual que se presenta al usuario tiene una alternativa textual
1.2.2 Subtítulos (grabados) A -> Se proporcionan subtítulos para el contenido audio grabado
1.3.1 Secuencia significativa A -> Cuando la secuencia en que se presenta el contenido afecta a su significado
1.1.1 -> Uso del atributo alt en el elemento img
1.2.2 -> Mantener subtítulos para cada contenido con audio
1.3.1 -> Añadir puntos de referencia como el parámetro role dentro de un 'div'
Operable
2.1 - Proporcionar acceso a toda la funcionalidad mediante el teclado
2.2 - Proporcionar a los usuarios el tiempo suficiente para leer y usar contenido
2.3 - Programas adapatados para cubrir espasmos, convusiones ...
2.4 - Proporcionar medios par ayudar a los usuarios a navegar y encontrarse en la web
2.1.1 Teclado A -> Toda la funcionalidad del contenido es operable a través de una interfaz de teclado sin que se requiera una determinada velocidad para cada pulsación individual de las teclas
2.2.1 Tiempo ajustable A -> Para cada límite de tiempo impuesto por el contenido, se cumple al menos uno de los siguientes casos: Apagar: El usuario puede detener el límite de tiempo antes de alcanzar el límite de tiempo . Ajustar ...
2.1.1 -> proporcionar controladores de eventos activador por teclado para la correcta navegación
2.2.1 -> Proporcionar una casilla de verificación en la primera página de un formulario de varias partes que permite a los usuarios solicitar un límite de tiempo de sesión más largo o ningún límite de tiempo de sesión.
Comprensible
3.1 - Hacer que los contenidos textuales resulten legibles y comprensibles
3.2 - Hacer que las páginas web aparezcan y operen de manera pedecible
3.3 - Ayudar a los usuarios a evitar y corregir los errores
3.1.1 Idioma de la página A -> El idioma predeterminado de cada página puede ser determinado por el software
3.1.3 Palabras inusuales A -> Se proporcionan mecanismos para identificar las definiciones específicas de palabras o frases
3.1.1 -> Usar el atributo de atributo lenguaje en el elemento html
3.1.3 -> Proporcionamos el significado de una palabra inusual o una frase específica
Robusto
4.1 - Maximizar la compatibilidad con las aplicaciones de usuarios actuales y futuras
4.1.1 Procesamiento A -> En los contenidos implementados mediante el uso de lenguajes de marcas, los elementos tienen las etiquetas de apertura y cierre completas; los elementos están anidados de acuerdo a sus especificaciones; los elementos no contienen atributos duplicados y los ID son únicos
4.1.2 Nombre, función, valor A -> Para todos los componentes de la interfaz de usuario (incluyendo pero no limitado a: elementos de formulario, enlaces y componentes generados por scripts), el nombre y la función pueden ser determinados por software; los estados, propiedades y valores que pueden ser asignados por el usuario pueden ser especificados por software
4.1.1 -> validación de páginas web: El objetivo de esta técnica es evitar ambigüedades en las páginas web que a menudo resultan de un código que no se valida con especificaciones formales
4.1.2 -> Usar aria-label para proporcionar una etiqueta invisible donde no se puede usar una etiqueta visible
